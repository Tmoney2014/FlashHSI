오늘(또는 지정 날짜)의 git 커밋 내역을 분석해 마크다운 작업 보고서를 생성한다.

## 사용법

```
/daily-report              # 오늘 작업 보고서 생성
/daily-report 2026-03-11   # 특정 날짜 보고서 생성
/daily-report yesterday    # 어제 보고서 생성
```

---

## Workflow

### Step 1: 커밋 내역 수집

```bash
# 오늘 커밋 전체 조회 (날짜 지정 없으면 today)
git log --oneline --since="today 00:00" --until="today 23:59" --format="%h %s"

# 커밋별 변경 파일 목록
git log --since="today 00:00" --until="today 23:59" --format="%h %s" --name-only

# 커밋 수 카운트
git log --oneline --since="today 00:00" --until="today 23:59" | wc -l
```

**$ARGUMENTS에 날짜가 있으면** 해당 날짜로 `--since`/`--until` 대체.  
예: `2026-03-11` → `--since="2026-03-11 00:00" --until="2026-03-11 23:59"`  
예: `yesterday` → `--since="yesterday 00:00" --until="yesterday 23:59"`

### Step 2: 커밋 내역 클러스터링

커밋들을 **주제 단위로 그룹화**한다 (커밋 메시지의 의미 기준):

- 같은 기능/모듈에 관련된 커밋들 → 하나의 "성과 블록"
- 독립적인 수정들 → 각각의 "성과 블록"
- 최대 3~5개 성과 블록으로 압축 (너무 세분화하지 않음)

### Step 3: 보고서 작성

아래 **정확한 포맷**으로 마크다운 보고서 생성:

```markdown
## 오늘의 주요 성과

**{성과 그룹 제목 — 핵심 기능/모듈명 중심}**

- **작업 내용:** {무엇을 했는지 — 명사형 나열}
- **사용자 혜택:** {이 작업으로 사용자/운영자가 얻는 이점}
- **기술 세부사항:**
    - {구체적 기술 변경 1}
    - {구체적 기술 변경 2}

**{성과 그룹 제목 2}**

- **작업 내용:** {무엇을 했는지}
- **사용자 혜택:** {사용자 이점}
- **주요 개선점:**
    - {개선 항목 1}
    - {개선 항목 2}

(성과 블록 반복...)

### 📊 작업 통계

- **커밋 횟수:** {N}회

### 🔧 기술적 개선사항

- ✨ {새 기능 또는 주요 추가 사항}
- 🚀 {성능/안정성 개선}
- 🔄 {수정 또는 변경}
- 🧹 {정리/리팩토링/메모리 관련}

### 💬 한 줄 요약

{오늘 작업 전체를 한 문장으로 — 가장 중요한 성과 중심}
```

---

### 작성 원칙

| 원칙 | 내용 |
|------|------|
| **독자 대상** | 비개발자도 이해 가능하게 (사용자 혜택 섹션) |
| **기술 정확성** | 기술 세부사항은 정확한 용어 사용 |
| **압축** | 같은 맥락의 커밋은 하나의 블록으로 묶음 |
| **이모지** | 🔧 섹션의 이모지만 사용 (✨🚀🔄🧹 등) |
| **성과 블록 제목** | 굵게(**) + 구체적 기능명 포함 |
| **분량** | 성과 블록 2~5개 적정 |

**`기술 세부사항` vs `주요 개선점`**:
- 새로 추가된 내용 → `기술 세부사항:`
- 기존 것을 개선/수정한 내용 → `주요 개선점:` 또는 `주요 작업:`

**사용자 혜택 작성법**:
- ❌ "코드 리팩토링으로 유지보수성 향상" (너무 기술적)
- ✅ "장시간 구동 시에도 프로그램 튕김이나 화면 깨짐 현상 억제"
- ✅ "복잡한 카메라 설정을 직관적으로 제어하고 오류 발생 시 대처 속도 향상"

### Step 4: 보고서 출력

생성된 마크다운을 **그대로 출력**한다 (코드블록으로 감싸지 않음).  
파일로 저장하지 않는다 — 사용자가 복사해서 사용.

## 보고서 예시 (참고용)

## 오늘의 주요 성과

**다중 관심 영역(MROI) 및 카메라 연동 고도화**

- **작업 내용:** 멀티 밴드 분석을 위한 MROI 설정 UI 추가 및 카메라 파라미터 추출 시스템 개선
- **사용자 혜택:** 복잡한 카메라 설정을 직관적으로 제어하고 오류 발생 시 대처 속도 향상
- **기술 세부사항:**
    - 분할 영역 집중 분석용 멀티 밴드 UI 개발
    - 카메라 설정 허용 범위 및 속성을 Params.txt로 출력하는 기능 개선

**영상 안정성 및 통신 최적화**

- **작업 내용:** 화면 멈춤 현상 방지를 위한 데이터 구조 교체 및 통신 모니터링 강화
- **사용자 혜택:** 장시간 구동 시에도 프로그램 튕김이나 화면 깨짐 현상 억제
- **주요 개선점:**
    - 레이스 컨디션 방지를 위한 라이브 영상(vizRow) 데이터 구조 개선
    - 점보 프레임 데이터 유실 방지 및 수신 상태 모니터링 로그 추가

### 📊 작업 통계

- **커밋 횟수:** 17회

### 🔧 기술적 개선사항

- ✨ 멀티 밴드 분석(MROI) 기반 마련
- 🚀 영상 처리 스레드 안정성 및 레이스 컨디션 해결
- 🔄 카메라 통신 패킷 유실 방지 및 로깅 강화
- 🧹 메모리 관리 최적화 및 UI 세부 레이아웃 정렬

### 💬 한 줄 요약

카메라 제어의 정밀도를 높이고 영상 출력의 치명적인 안정성 문제를 해결
